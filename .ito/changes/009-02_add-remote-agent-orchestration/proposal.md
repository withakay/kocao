## Why

Module 007 gave Kocao a way to launch and interact with one sandbox-backed agent session at a time, but users still have to manage raw run IDs and ad-hoc workflows manually. Once lifecycle reliability from `009-01` is in place, the next leverage point is a first-class orchestration layer that lets Kocao coordinate named remote agents, durable task dispatch, transcripts, and operator-facing visibility.

## What Changes

- Add named remote agents, teams, and pools so users can target durable logical workers instead of raw run IDs.
- Add task dispatch semantics for assigning work to a remote agent and collecting structured results.
- Persist transcripts, summaries, and generated artifacts so sessions produce reusable outputs instead of transient logs only.
- Add multi-agent workflow support for reviewer/implementer/researcher patterns built on top of module 007's sandbox-agent sessions rather than replacing them.
- Add a lightweight remote-agent dashboard so operators can see active agents, current tasks, last activity, and artifacts.
- Keep this change layered on top of module 007 and `009-01`; it should not introduce a second transport or bypass the sandbox-agent session contract.

## Capabilities

### New Capabilities

- `remote-agent-orchestration`: named agents, pools, task dispatch, retries, cancellation, and result collection.
- `agent-artifacts`: persistent transcripts, result summaries, and attached artifacts generated by remote agents.

### Modified Capabilities

- `control-plane-api`: orchestration endpoints and task/result lifecycle semantics.
- `web-ui`: dashboard and operator-facing views for active remote agents, tasks, and outcomes.
- `session-durability`: persist orchestration metadata, transcripts, and artifact references alongside session state.

## Impact

- Affected code: `internal/controlplaneapi/*`, `internal/controlplanecli/*`, `cmd/kocao/*`, `web/src/ui/*`, durability/storage code, and related tests/docs.
- APIs: new orchestration resources for named agents, task dispatch, task status, transcripts, and artifacts.
- Operations: moves Kocao from single-session control toward coordinated remote-agent operations built on the stable session model from module 007 and `009-01`.
- Module relationship: depends on the sandbox-agent runtime contract from module 007 and the lifecycle guarantees introduced by `009-01_harden-agent-session-lifecycle`.
